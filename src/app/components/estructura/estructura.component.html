Flujo de arranque

      1. En el index.html tenemos un componente raíz de la aplicación "app-root".
      2. El archivo por el que Webpack comienza a producir los bundles es el main.ts
      3. Al servir la aplicación (ng serve), o al llevarla a producción (ng build) la herramienta Webpack genera los paquetes (bundles) de código del proyecto y coloca los correspondientes scripts en el index.html, para que todo funcione.
      4. Dentro del main.ts encontramos un import del módulo principal (AppModule) y la llamada al sistema de arranque (bootstrapModule) en la que pasamos por parámetro el módulo principal de la aplicación.
      5. En el módulo principal se importa el componente raíz (AppComponent) y en el decorador @NgModule se indica que este componente forma parte del bootstrap.
      6. El código del componente raíz tiene el selector "app-root", que es la etiqueta que aparecía en el index.html.
      7. El template del componente raíz, contiene el HTML que se visualiza al poner en marcha la aplicación en el navegador.
Referencia

Estructura del proyecto

{{ node.data.name }}

e2e

En esta carpeta se colocan los archivos para la realización de las pruebas "end to end". Nunca lo he utilizado

node_modules

Es la carpeta donde npm va colocando todas las dependencias del proyecto, es decir, el código de todas las librerías o componentes que estemos usando para basarnos en el desarrollo de una aplicación. Por ejemplo, el propio Angular es una dependencia.

dist

Es la carpeta donde donde se colocarán los archivos distribuibles para publicar una aplicación. Super fácil: básicamente --> ng build.

src

Es la carpeta donde están las fuentes del proyecto. Esta carpeta es la que usaremos para desarrollar la aplicación y donde iremos colocando componentes y otro tipo de artefactos necesarios para poner en marcha nuestras ideas.

.angular-cli.json

Entre otras cosas se pueden meter aquí los scripts y css que serán incluidos en el build. Referencia completa

package.json

Aquí aparecen las dependencias del proyecto. Cuando instalas algo con npm aparece reflejado aquí.

También se pueden personalizar y explicitar aquí los comandos de angular ng

Ejemplo

index.html

Es el comienzo en html de toda la aplicación.

Aquí se pueden meter los scripts y css necesarios, aunque es mejor hacerlo en .angular-cli.json

En el body nos encontramos con el siguiente selector de componente app-root. Esto muestra el componente raíz cuyo selector es app-root

main.ts

Webpack, que es quien compila y transpila toda la aplicación, se encarga en primer lugar de ejecutar este archivo.

Es el punto de arranque, se importan varias librerías externas necesarias y nuestro componente de inicio, que es el AppModule. Después se lanza con la instrucción:

platformBrowserDynamic().bootstrapModule(AppModule);

index.html

Aquí se meten todos los archivos externos que necesitemos como iconos, imagenes,...

favicon.ico

Pues el icono de la aplicación

styles.css

La hoja de estilos principal de la aplicación. Cada componente puede tener una específica si se quiere que prevalece sobre esta.

Esta hoja de estilos es referenciada en el archivo angular.cli.json y ahí se establece el orden de carga respecto al resto de hojas de estilo, como por ejemplo, la de bootstrap

app

Aquí es donde se mete toda la chicha.

components

Aquí es donde se meten los componentes de la aplicación.

services

Aquí es donde se meten los servicios de la aplicación.

interfaces

Aquí es donde se meten las interfaces de la aplicación.

pipes

Aquí es donde se meten los pipes de la aplicación.

app.module.ts

De entre todos los imports que se hace en el main.ts hay uno fundamental, que nos sirve para tirar del hilo y ver qué es lo que está pasando por abajo, para hacer posible que todo comience a funcionar. Se trata de app.module.ts, que podemos considerar como el módulo principal de la aplicación.

Encontramos el import al componente raíz de la aplicación (esa etiqueta HTML no-estándar que aparecía en el BODY del index).

import AppComponent from './app.component';

app.component.html

Equivale a lo que conocemos por "vista" en la arquitectura MVC.

Aquí puedes meter cualquier tipo de código html y puedes llamar a otros componentes.

app.component.ts

Equivale a lo que conocemos por "controlador" en la arquitectura MVC.

En el decorador Component está 'selector', templateUrl:'./app.component.html', stylesUrls:['./app.component.css']

app.component.css

Aquí se pueden meter los estilos específicos del componente.

app.component.spec.ts

Archivo para pruebas.
